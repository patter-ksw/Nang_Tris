# 냥트리스 프로젝트 개발 규칙 (Nang_Tris Project Rules)

## 진행 프로세스 규칙
1. **계획서 작성 및 승인**:
   - 새로운 기능이나 변경 사항을 개발하기 전, `implementation_plan.md`를 작성하여 1차 진행 계획을 수립합니다.
   - 이때 `implementation_plan.md`의 `ArtifactMetadata`에서 `RequestFeedback`은 항상 `false`로 설정합니다. (사용자가 UI 상에서 별도로 승인(Proceed) 버튼을 클릭할 필요가 없도록 하기 위함입니다.)
   - 채팅창을 통해 사용자에게 구현 계획의 핵심 요약을 간단히 전달합니다.
2. **시작 신호 접수**:
   - 사용자가 "시작해줘" 또는 이와 유사한 동의 표현을 남기면, 즉시 계획서에 따라 구현 및 배포 검증을 시작합니다.
3. **자동 배포 확인 및 진행**:
   - 개발 및 로컬 검증이 끝나면 Git 커밋 후 푸시하여 Vercel 배포를 자동으로 진행하고 완료 상태까지 직접 확인합니다.
   - 과정 중에 추가로 확인하거나 질문할 필요 없이, 스스로 판단하여 최종 검증까지 완료한 뒤 결과를 보고합니다.
